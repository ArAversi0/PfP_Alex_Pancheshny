# Этап 03 — Архитектурное проектирование

## Обзор

На данном этапе была выполнена разработка архитектурного каркаса проекта **«Система сопровождения персонажей настольно-ролевой системы PFP»**.

Основной целью этапа являлось преобразование результатов бизнес-анализа и системного анализа, выполненных на предыдущих этапах, в формальную архитектурную модель, определяющую структуру будущей программной системы, правила взаимодействия компонентов и принципы дальнейшей реализации.

В качестве базового архитектурного подхода был выбран паттерн **PCMEF (Presentation – Control – Mediator – Entity – Foundation)**, адаптированный под enterprise-конфигурацию проекта.

---

# Цели этапа

В рамках архитектурного проектирования решались следующие задачи:

- определение архитектурного стиля системы;
- адаптация PCMEF под выбранную enterprise-траекторию;
- определение границ системы и внешних интеграций;
- проектирование архитектурных слоёв;
- определение зависимостей между компонентами;
- спецификация интерфейсов взаимодействия;
- проектирование схемы развёртывания;
- подготовка архитектурной базы для перехода к реализации.

---

# Архитектурная концепция

Проект реализуется как **enterprise-oriented companion platform**, предназначенная для сопровождения персонажей настольно-ролевой системы PFP.

Архитектура построена на принципах:

- разделения ответственности;
- слабой связанности компонентов;
- высокой тестируемости;
- централизации бизнес-логики;
- независимости пользовательских интерфейсов от серверной части.

В качестве основной архитектурной модели используется **Modular Monolith**, реализованный на базе Spring Boot.

---

# Выбранная траектория

Для проекта выбрана траектория:

```text
Enterprise
```

Конфигурация системы включает:

```text
Web Client (React + TypeScript)
            │
            ▼
        REST API
            ▲
            │
Desktop Client (JavaFX)
            │
            ▼
      PostgreSQL
```

Такой подход позволяет поддерживать несколько типов клиентских приложений при сохранении единого централизованного backend.

---

# Реализация PCMEF

В рамках проекта слои PCMEF распределены следующим образом.

| Слой | Реализация |
|--------|--------|
| Presentation | React web-клиент, JavaFX desktop-клиент |
| Control | REST-контроллеры Spring Boot |
| Mediator | Сервисный слой |
| Entity | Предметная модель, JPA-сущности |
| Foundation | Репозитории Spring Data |
| Security | JWT, OAuth2, Spring Security |
| DTO | API-контракты |
| Local Cache | Локальное desktop-хранилище |

Архитектурная модель обеспечивает строго направленные зависимости сверху вниз и исключает появление циклических связей между слоями.

# Основные архитектурные решения

На текущем этапе были зафиксированы следующие ключевые решения.

## ADR-001

Использовать архитектурный стиль **Modular Monolith** вместо микросервисной архитектуры.

Причины:

- меньшая инфраструктурная сложность;
- соответствие масштабу проекта;
- упрощённое сопровождение.

---

## ADR-002

Использовать **REST API** как единую точку взаимодействия клиентов и серверной части.

Причины:

- независимость клиентов;
- стандартизированное взаимодействие;
- удобство масштабирования.

---

## ADR-003

Использовать **PostgreSQL** как основную систему хранения данных.

Причины:

- поддержка ACID;
- надёжность;
- интеграция со Spring Data JPA.

---

## ADR-004

Использовать **JWT + OAuth2** для аутентификации и авторизации.

Причины:

- безопасность API;
- поддержка внешних провайдеров авторизации;
- централизованное управление доступом.
