# 05-design — Детальное проектирование поведения системы

## Общая информация

На данном этапе выполнено детальное проектирование поведения системы
«Система сопровождения персонажей настольно-ролевой системы PFP».

Основная цель этапа — переход от архитектурного уровня (ЛР3) и модели данных (ЛР4)
к спецификации поведения системы и подготовке к реализации (ЛР6).

---

## Цели этапа

- формализация ключевых сценариев использования;
- построение диаграмм последовательностей;
- уточнение взаимодействия слоёв PCMEF;
- детализация бизнес-логики;
- подготовка к реализации backend-ядра;
- выявление ключевых методов и интерфейсов.

---

## Выполненные работы

В рамках этапа были выполнены:

### 1. Выбор ключевых сценариев

Были выбраны критические сценарии системы:

- создание персонажа;
- управление инвентарём и экипировкой;
- импорт и экспорт персонажа.

Выбор основан на их высокой нагрузке на бизнес-логику и межслойное взаимодействие.

---

### 2. Построение диаграмм последовательностей

Для каждого сценария построены диаграммы последовательностей, отражающие:

- поток выполнения операций;
- распределение ответственности между слоями;
- точки валидации;
- взаимодействие с persistence-слоем.

Диаграммы строго соответствуют архитектуре:

Presentation → Control → Mediator → Entity → Foundation

---

### 3. Уточнение модели классов

На основе диаграмм поведения была уточнена структура:

- сервисов;
- контроллеров;
- репозиториев;
- агрегатов домена.

Добавлены:

- недостающие методы;
- интерфейсы сервисов;
- зависимости между компонентами.

---

### 4. Спецификация бизнес-логики

Были формализованы:

- методы сервисов;
- операции доменных объектов;
- правила валидации;
- сценарии ошибок.

---

### 5. Применение паттернов проектирования

В архитектуру были интегрированы GoF паттерны:

- Strategy — для расчёта характеристик;
- Factory Method — для создания объектов импорта/экспорта;
- Builder — для формирования агрегата Character.

Это позволило:

- повысить гибкость системы;
- снизить связанность;
- улучшить расширяемость.

---
