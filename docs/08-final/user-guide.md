# Руководство пользователя

## 1. Назначение приложения

PfP Companion System помогает вести персонажей собственной настольной ролевой системы PfP (Pain for Pleasure). Приложение доступно в двух вариантах:

- веб-версия;
- desktop-версия.

Веб-версия работает через браузер и backend. Desktop-версия может работать локально в гостевом режиме или синхронизироваться с backend в режиме аккаунта.

## 2. Веб-версия

### Вход

Пользователь может войти:

- по email и паролю;
- через Google.

При обычной регистрации на почту отправляется письмо подтверждения. В development-режиме письмо можно открыть в Mailpit.

### Архив персонажей

После входа пользователь попадает в архив персонажей. На странице отображаются:

- карточки персонажей;
- счётчик `N / 100`;
- кнопка создания нового персонажа;
- кнопка импорта JSON.

По карточке можно открыть лист персонажа. Персонажа можно удалить. При создании нового персонажа приложение открывает его лист.

### Лист персонажа

Лист персонажа содержит:

- портрет;
- основную информацию;
- уровень и боевые/игровые показатели;
- здоровье по частям тела;
- характеристики и навыки;
- экипировку;
- инвентарь;
- заклинания;
- дополнительные заметки.

Для редактирования нужно нажать `Edit Sheet`. После изменения данных нужно нажать `Save Changes`.

### Инвентарь

В инвентаре можно:

- добавить предмет кнопкой Add item;
- добавить предмет кликом по пустой ячейке;
- редактировать предмет;
- перемещать предметы drag-and-drop;
- экипировать предметы подходящего типа;
- снять экипировку;
- продать trade-предмет;
- выбросить предмет.

Ячейки инвентаря идут по 10 в строке. Изображение предмета не должно менять размер ячейки.

### Заклинания

В блоке Spells можно создавать и редактировать заклинания. Заклинание может иметь изображение, которое отображается в карточке.

### Импорт и экспорт JSON

Экспорт создаёт переносимую JSON-копию персонажа. Импорт позволяет создать персонажа из JSON-файла. Это полезно для резервного копирования и переноса персонажа между окружениями.

### Lore и Rule book

В верхнем меню доступны:

- `Lore`;
- `Rule book`.

Обе страницы позволяют читать категории и статьи. В текущем MVP поиск не реализован.

## 3. Desktop-версия

### Запуск

Во время разработки desktop запускается командой:

```powershell
$env:JAVA_HOME="C:\Users\user\.jdks\corretto-23.0.2"
.\gradlew.bat :apps:desktop:run
```

После сборки app image можно запускать:

```text
apps/desktop/build/jpackage/image/PfP Companion/PfP Companion.exe
```

### Главное меню

На главном экране доступны:

- Characters;
- Extra Content;
- Settings;
- Exit.

### Гостевой режим

Гостевой режим работает без backend и хранит персонажей локально. Лимит гостевого режима - 30 персонажей.

В гостевом режиме доступны:

- создание персонажа;
- открытие персонажа;
- редактирование;
- импорт JSON;
- экспорт JSON;
- удаление локального персонажа.

### Режим аккаунта

Режим аккаунта требует запущенный backend и вход в аккаунт. Персонажи загружаются из того же аккаунта, который используется в веб-версии.

В режиме аккаунта доступны:

- просмотр синхронизированных персонажей;
- создание персонажа;
- импорт JSON с синхронизацией на backend;
- экспорт JSON;
- редактирование листа;
- удаление персонажа.

Если backend недоступен, приложение должно показать понятную ошибку подключения.

### Desktop-лист персонажа

Desktop-лист повторяет основные блоки веб-листа:

- Character Portrait;
- Character Info;
- Level / Defence / Dodge / Blessings / Inspirations / Movement / Health / Money;
- Stats and Skills;
- Equipment;
- Condition;
- Inventory;
- Spells;
- Additional Info.

Внизу окна закреплена панель действий. Она позволяет быстро сохранить лист, выйти из режима редактирования, экспортировать JSON или вернуться к архиву.

### Дополнительный контент

В desktop доступны Lore и Rule book в режиме только для чтения. Для MVP они встроены в приложение и одинаковы для гостевого режима и режима аккаунта.

### Настройки

Раздел настроек содержит:

- General - разрешение окна и режим отображения;
- Controls - горячие клавиши;
- Music and Sounds - выбор музыки и громкости.

Горячие клавиши можно переназначить. Музыку и звуки можно уменьшить до 0%.

## 4. Рекомендации

- Для постоянной игры лучше использовать режим аккаунта, чтобы персонажи синхронизировались между веб-версией и desktop.
- Для быстрых локальных тестов можно использовать гостевой режим.
- Перед экспериментами с крупными изменениями персонажа желательно экспортировать JSON.
- Для проверки регистрации в development-режиме нужно открыть Mailpit.
- Если desktop-режим аккаунта не входит в аккаунт, сначала проверьте, запущен ли backend.

## 5. Частые проблемы

### Не приходит письмо подтверждения

В development-режиме письмо не приходит в реальный почтовый ящик. Его нужно смотреть в Mailpit:

```text
http://localhost:8025
```

### Desktop не синхронизируется

Проверьте:

- запущен ли backend;
- доступен ли `http://localhost:8080`;
- выполнен ли вход в режим аккаунта;
- не истекла ли сессия.

### Не создаётся установщик

Для Windows-установщика нужен WiX Toolset. Если WiX не установлен, используйте self-contained app image.

### Персонаж не помещается на экране

В Settings выберите подходящее разрешение. Приложение поддерживает рабочие варианты от 1024x640 до 1920x1080.
