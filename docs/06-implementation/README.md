# Реализация

Раздел описывает фактическую реализацию PfP Companion System после завершения веб-части и desktop-части. Проект рассматривается как enterprise-приложение с несколькими клиентами, общим доменным ядром и централизованным backend API.

## Общая схема

В проекте выделены четыре основные части:

- `apps/backend` - серверное приложение Spring Boot.
- `apps/web` - веб-клиент на React.
- `apps/desktop` - desktop-клиент на JavaFX.
- `libs/game-rules` - общий модуль правил PfP, используемый клиентскими и серверными сценариями.

Backend отвечает за пользователей, авторизацию, персонажей, импорт/экспорт, контент лора и справочника правил. Веб-клиент предоставляет основной online-интерфейс. Desktop-клиент поддерживает два режима: гостевой режим с локальными JSON-файлами и режим аккаунта с синхронизацией через backend API.

## Backend

Backend реализован на Spring Boot и разделён по доменным модулям:

- `identityaccess` - регистрация, вход, Google OAuth2, JWT, пользователи, роли, подтверждение email.
- `charactersheet` - создание, чтение, редактирование и удаление листов персонажей.
- `charactertransfer` - импорт и экспорт листов персонажей в JSON.
- `content` - лор и справочник правил, включая админское редактирование.
- `notification` - отправка писем через SMTP/Mailpit.
- `sharedkernel` - общие типы и исключения.

Для persistence-слоя используется Spring Data JPA и PostgreSQL. Миграции схемы выполняются через Flyway. Валидация входных DTO построена на Jakarta Validation. Ошибки API приводятся к человекочитаемому формату через общий обработчик исключений.

## Web

Web-клиент реализован на React, TypeScript и Vite. Основные feature-модули:

- `auth` - страницы входа, регистрации, восстановления пароля, подтверждения email, Google OAuth2 callback.
- `character-sheet` - архив персонажей, лист персонажа, guest-хранилище, игровые блоки листа.
- `character-transfer` - JSON import/export.
- `content` - страницы Lore и Rule book.
- `admin` - админская панель, пользователи, персонажи пользователей, редактирование контента.

Веб-лист персонажа содержит портрет, характеристики, здоровье, экипировку, инвентарь, заклинания и дополнительные сведения. Для инвентаря реализованы ячейки по 10 слотов в строке, drag-and-drop, создание предмета из пустой ячейки, продажа trade-предметов, экипировка и снятие предметов.

## Desktop

Desktop-клиент реализован на JavaFX. Он повторяет основные пользовательские сценарии веб-версии, но адаптирован под desktop-компоновку:

- главное меню;
- выбор режима персонажей;
- гостевой архив;
- архив аккаунта;
- лист персонажа;
- импорт и экспорт JSON;
- Lore и Rule book в режиме только для чтения;
- настройки;
- музыка, звуки и горячие клавиши.

Гостевой режим хранит персонажей локально в JSON и ограничен 30 листами. Режим аккаунта работает с backend API и использует тот же лимит 100 листов, что и веб-версия. Созданные или импортированные персонажи аккаунта синхронизируются с backend.

В desktop-листе реализованы игровые блоки веб-версии: Character Info, сводные параметры, Stats and Skills, Equipment, Condition, Inventory, Spells и Additional Info. Для удобства редактирования используется закреплённая нижняя панель действий.

## Дополнительный контент

Лор и справочник правил в веб-версии загружаются из backend и могут редактироваться администратором. В desktop-версии для MVP используется встроенный контент из ресурсов приложения в режиме только для чтения:

- `apps/desktop/src/main/resources/com/pfp/desktop/content/lore.json`
- `apps/desktop/src/main/resources/com/pfp/desktop/content/rules.json`

Синхронизация дополнительного контента с backend в MVP не включалась намеренно: для гостевого режима и режима аккаунта desktop-клиента используется одинаковый встроенный справочник.

## Сборка desktop-приложения

Для desktop-приложения подготовлена сборка через `jpackage`.

Self-contained app image:

```powershell
.\gradlew.bat :apps:desktop:packageDesktopAppImage --console=plain
```

Windows-установщик:

```powershell
.\gradlew.bat :apps:desktop:packageDesktopInstaller --console=plain
```

App image работает без отдельной установки Java, так как содержит runtime. Для Windows installer требуется WiX Toolset.
