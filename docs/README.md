## Бизнес-контекст

Игроки и мастера собственной НРИ-системы PfP сталкиваются с тем, что лист персонажа, инвентарь, заклинания, здоровье по частям тела, лор и справочник правил приходится вести в разных местах. Это повышает риск ошибок, затрудняет перенос данных между устройствами и мешает быстро обновлять персонажа во время игры.

PfP Companion System решает эту проблему через единую companion-платформу: web-клиент используется как основное online-приложение, desktop-клиент поддерживает локальную работу и синхронизацию с аккаунтом, backend хранит пользователей, персонажей и контент, а общий модуль правил помогает удерживать игровую логику в одном виде.

## Цели проекта

1. Реализовать полноценный лист персонажа PfP с поддержкой игровых блоков, инвентаря, экипировки, заклинаний и заметок.
2. Обеспечить web-доступ и desktop-доступ к персонажам с синхронизацией через аккаунт.
3. Поддержать локальный гостевой режим в desktop-приложении без обязательного подключения к backend.
4. Реализовать админский контур для пользователей, персонажей и контента.
5. Подготовить проектную документацию по этапам жизненного цикла ПО.
